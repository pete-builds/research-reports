## 6. Hiring Process

### Overview

The hiring process averages **14-20 days** from application to offer, based on Glassdoor user-reported data. [source: glassdoor.com/Interview/Anthropic (US: 19 days/117 interviews; UK: 16 days/22 interviews; HK: 14 days/3 interviews)]

### Interview Stages

**Stage 1: Application**
Submit via Anthropic's Greenhouse-powered careers page. ATS-friendly resume recommended. Technical roles benefit from GitHub/portfolio links. [source: finalroundai.com/blog/anthropic-interview-process]

**Stage 2: Recruiter Screen (20-30 minutes)**
Background, motivation, baseline qualifications. Focus on communication skills, enthusiasm, and cultural alignment. [source: finalroundai.com]

**Stage 3: Online Technical Assessment**
Format varies by role. Software engineers face coding challenges; researchers face ML problems. Can be time-boxed (hours to days). For strong candidates, this step may be skipped. [source: finalroundai.com, hellointerview.com]

**Stage 4: Hiring Manager Call (45-60 minutes)**
Deeper technical discussion, take-home review, design trade-offs. [source: finalroundai.com]

**Stage 5: Final Interview Loop (half-day to full day)**
Multiple rounds covering coding, system design, ML fundamentals, behavioral/culture-fit. For SWE Infrastructure roles, one Reddit user reported **5 rounds over 3 weeks**: online assessment (90 min, 2 problems including LRU Cache in Python), then onsite rounds including coding, system design, project deep dive, and HM interview. [source: reddit.com/r/theprimeagen, February 2026; hellointerview.com]

**Stage 6: Reference Checks and Offer**
Background verification followed by offer. [source: igotanoffer.com]

### Key Details

- **Interview platform:** Google Meet (timezone-flexible). Technical: Colab/CodeSignal. [source: anthropic.com/careers]
- **AI usage policy:** Candidates may use AI tools for preparation but not during live interviews or assessments. Anthropic itself uses Claude for job descriptions, interview questions, candidate communications, and sourcing. They do not let Claude make hiring decisions. [source: anthropic.com/candidate-ai-guidance]
- **Re-application:** Allowed after 12 months [source: anthropic.com/careers]
- **Internships:** Not currently offered [source: anthropic.com/careers]
- **Fellows Program:** Next cohort starts July 20, 2026. Apply by April 26, 2026. 80%+ of fellows produce papers; 25-50% receive full-time offers. [source: job-boards.greenhouse.io/anthropic/jobs/5023394008]
- **Skills over credentials:** "About half our technical staff had no prior ML experience" [source: anthropic.com/careers]

#### Option B: Full Tracker (Python + SQLite + Dashboard)

```python
# Core scraper (runs daily via cron)
import httpx, sqlite3, json
from datetime import datetime

GREENHOUSE_API = "https://boards-api.greenhouse.io/v1/boards/anthropic/jobs"

def fetch_jobs():
    resp = httpx.get(GREENHOUSE_API)
    data = resp.json()
    return data["jobs"]

def snapshot(jobs, db):
    ts = datetime.now().isoformat()
    for job in jobs:
        db.execute("""
            INSERT INTO job_snapshots 
            (timestamp, job_id, title, department, location, updated_at)
            VALUES (?, ?, ?, ?, ?, ?)
        """, (ts, job["id"], job["title"],
              job.get("departments", [{}])[0].get("name", "Unknown"),
              job.get("location", {}).get("name", "Unknown"),
              job.get("updated_at")))
    db.commit()

# Schema
"""
CREATE TABLE job_snapshots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    job_id INTEGER NOT NULL,
    title TEXT,
    department TEXT,
    location TEXT,
    updated_at TEXT
```
